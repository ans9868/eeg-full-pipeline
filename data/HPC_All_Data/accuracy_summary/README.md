# Official PCA and ANOVA Accuracy Summary

**No cross-experiment aggregation.** Every statistic is computed only over runs
**within** a single experiment, at the chosen abstraction level
(experiment, experiment x model, or experiment x model x hyperparams).

**Source data:** `../all_experiments_combined.csv` (10 canonical experiments, 2,736 rows).

---

## 1. Scope: Which Experiments

All 10 canonical experiments are included. Use `experiment_type` and `feature_set`
columns to filter as needed:

| Scope | Experiments |
|-------|-------------|
| LPSO Random-50 | `ANOVA_L_6_Random`, `ANOVA_L_2_Random`, `PCA_L_6_Random`, `PCA_L_2_Random` |
| LPSO Systematic-12 | `ANOVA_L_6_Uniform`, `PCA_L_6_Uniform` |
| Within-subject | `ANOVA_W_C`, `ANOVA_W_F`, `PCA_W_C`, `PCA_W_F` |

---

## 2. Two Statistics (Both in Separate Columns)

Where a central tendency is reported, **two** definitions appear side-by-side:

| Column | Meaning |
|--------|---------|
| `median_all_runs` | Median over **all** runs in the group (folds x HP configs). Conservative; matches booklet Tables B/D convention. |
| `median_best_per_fold` | For each fold, take the **max** accuracy in the group; then median of those per-fold bests. |

Other stats (IQR, mean, SD, min, max) are computed on the **all-runs** pool.

---

## 3. Output CSVs

### 3.1 `per_experiment_per_model_stats.csv`

One row per **(experiment, model)**.
Within each experiment and model: all runs (folds x HP configs) are pooled.

Columns: `experiment`, `experiment_dir`, `experiment_type`, `feature_set`,
`holdout_size_P`, `model`, `n_runs`, `n_folds`,
`median_all_runs`, `median_best_per_fold`,
`iqr_all_runs`, `mean_all_runs`, `sd_all_runs`, `min_all_runs`, `max_all_runs`.

Generated by: `01_per_experiment_per_model_stats.py`

### 3.2 `per_experiment_best_model_stats.csv`

One row per **experiment**.
Best model = model with highest `median_all_runs` in that experiment
(ties broken alphabetically).

Columns: `experiment`, `experiment_type`, `feature_set`, `holdout_size_P`,
`best_model`, `n_runs`, `n_folds`,
`median_all_runs`, `median_best_per_fold`,
`iqr_all_runs`, `mean_all_runs`, `sd_all_runs`, `min_all_runs`, `max_all_runs`.

Generated by: `02_per_experiment_best_model_stats.py`

### 3.3 `per_experiment_per_model_per_hyperparam_stats.csv`

One row per **(experiment, model, hyperparameters)**.
At this level there is exactly one run per fold, so "all runs" and
"best per fold" are the same; a single set of fold-level stats is reported.

Columns: `experiment`, `experiment_type`, `feature_set`, `holdout_size_P`,
`model`, `hyperparams`, `n_folds`,
`median`, `iqr`, `mean`, `sd`, `min`, `max`.

Generated by: `03_per_experiment_per_model_per_hyperparam_stats.py`

### 3.4 `per_experiment_all_models_pooled_stats.csv`

One row per **experiment**.
All models and HP configs within that experiment are pooled together.

Columns: `experiment`, `experiment_type`, `feature_set`, `holdout_size_P`,
`n_runs`, `n_folds`,
`median_all_runs`, `median_best_per_fold`,
`iqr_all_runs`, `mean_all_runs`, `sd_all_runs`, `min_all_runs`, `max_all_runs`.

Generated by: `04_per_experiment_all_models_pooled_stats.py`

## 4. How to Run

```bash
cd data/HPC_All_Data/accuracy_summary
for f in 0*.py; do python "$f"; done
```

Scripts read `../all_experiments_combined.csv` and write their CSV into this
directory. They do not modify the source CSV.

---

## 5. Design Rules

1. **No cross-experiment aggregation.** Every row belongs to exactly one experiment.
2. **Two median columns** where applicable (`median_all_runs` + `median_best_per_fold`).
3. **Transparent scope.** `experiment_type` and `feature_set` let you filter
   to LPSO-only or feature-specific subsets in downstream work.
