## Common Issues and Troubleshooting

### File System Access Issues on HPC Systems

**Problem**: Singularity jobs fail with "no such file or directory" errors even though files exist on the login node.

**Cause**: On HPC systems, login nodes and compute nodes may have different file system access. Files in certain directories (like `/archive/`) may not be accessible from compute nodes where SLURM jobs run.

**Symptoms**:
- Direct `singularity run` commands work on login node
- SLURM jobs fail with file not found errors
- Error messages like: `WARNING: skipping mount of /archive/...: stat /archive/...: no such file or directory`

**Solutions**:

1. **Copy files to shared filesystem** (Recommended):
   ```bash
   # Copy data to /scratch/ which is accessible on both login and compute nodes
   mkdir -p /scratch/username/eeg-data
   cp /archive/username/data/*.set /scratch/username/eeg-data/
   ```
   Then update your config file to use the new paths.

2. **Use the automatic data copying feature**:
   The pipeline now automatically copies data files to `/scratch/` when using "Singularity with Slurm" deployment method.

3. **Contact HPC administrators**:
   - Ask about file system access differences between login and compute nodes
   - Inquire about recommended approaches for accessing archive data from SLURM jobs
   - Check if specific directories are accessible from compute nodes

**Prevention**: Always test file access from compute nodes before running large jobs:
```bash
# Test file access from compute node
cat > test_access.slurm << 'EOF'
#!/bin/bash
#SBATCH --nodes=1 --cpus-per-task=1 --mem=1G
#SBATCH --job-name=test-access
ls -la /path/to/your/data/files
EOF
sbatch test_access.slurm
```

### Docker Permission Issues

**Problem**: Docker commands fail with "operation not permitted" errors.

**Solutions**:
- On macOS: Go to System Preferences > Security & Privacy > Privacy > Full Disk Access and add Docker
- On Linux: Ensure Docker has permission to access mounted directories
- Check that all paths in your config file exist and are accessible

### Singularity Build Issues

**Problem**: Singularity container builds fail.

**Solutions**:
- Ensure Singularity is installed and available in PATH
- Check build logs in `./containers/` directory
- Verify Docker images exist and are accessible
- On HPC systems, use SLURM for building containers if local builds fail
